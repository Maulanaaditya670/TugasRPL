Tugas 2 B Pemeliharaan Mesin ATM

  • Judul Tugas: Pemeliharaan Mesin ATM
  • Deskripsi: Membuat usecase,activity diagram tentang Pemeliharaan Mesin ATM
  1. Analisis Kebutuhan

    2. Kebutuhan fungsional

    • Sistem harus mampu mencatat dan memonitor pemeliharaan ATM secara real-time.
    • Sistem harus memiliki fungsi untuk melaporkan gangguan ATM secara otomatis.
    • Sistem harus mengelola jadwal pemeliharaan dan perbaikan ATM secara efisien.
    • Sistem harus dapat mencatat proses penggantian komponen atau perbaikan yang dilakukan.
    • Sistem harus menghasilkan laporan status pemeliharaan ATM secara otomatis setelah data dimasukkan ke database.

    Kebutuhan Fungsional

    • Sistem harus memproses validasi laporan gangguan dan pembuatan laporan dalam waktu cepat.
    • Sistem harus melindungi seluruh data baik data pemeliharaan maupun data jadwal perbaikan.
    • Sistem harus memiliki uptime sebesar 99,9% agar selalu siap digunakan dalam operasional.
    • Laporan harus akurat, real-time, dan tersedia dalam format yang mudah dipahami (PDF, Excel, atau digital dashboard).

  2. Desain Sistem

    3. Arsitektur Sistem

    • Divisi Rescheduler: Antarmuka untuk mengajukan laporan gangguan dan permintaan pemeliharaan.
    • Divisi Administrasi: Antarmuka untuk memvalidasi laporan gangguan, menginput data hasil pemeliharaan, dan memonitor jadwal pemeliharaan..
    • Pimpinan: Antarmuka untuk melihat laporan status pemeliharaan secara real-time dan visualisasi data (grafik, tabel).
    • Bank: Antarmuka untuk menerima laporan status ATM dalam format terstruktur (PDF, Excel, dll.).
    • Server: Untuk menyimpan data laporan dan jadwal pemeliharaan di database

  3. Alur Proses
    • Laporan Gangguan:

      Scheduler melaporkan gangguan ATM kepada teknisi.

    • Verifikasi:

      Teknisi memvalidasi laporan gangguan ATM. Jadwal

    • Proses Pemeliharaan:

      Teknisi mencatat proses perbaikan atau penggantian komponen ATM.

    • Laporan Status ATM:

      Pimpinan dan Bank menerima laporan status pemeliharaan ATM.

  5. Implementasi

    6. Teknologi bahasa pemrograman yang digunakan yaitu :
    Frontend

    • Framework: React.js (untuk Web) atau Flutter (untuk Mobile).
    • Styling: Bootstrap CSS

    Backend

    • Bahasa Pemrograman: Node.js dengan Express.js atau Python
    • RESTful API untuk komunikasi antara front-end dan back-end

    Database
    Database yang digunakan yaitu mysql

    Keamanan
    Autentikasi: JWT (JSON Web Token)

  6. Pengujian Sistem

    7. Pengujian dilakukan untuk memastikan semua fitur dan alur sudah sesuai. Pengujian ini menggunakan blackbox testing untuk menguji fitur yang telah dibuat. Jika ditemukan bug, maka sistem dilakukan perbaikan.

  7. Deployment

    8. Tahap deployment Sistem Pemeliharaan ATM menggunakan arsitektur Client-Server, di mana front-end dibangun dengan React.js, sementara back-end menggunakan Node.js yang terhubung ke database MySQL. Proses ini mencakup instalasi dependensi, konfigurasi database, dan pengaturan lingkungan produksi. Sistem diuji untuk memastikan fungsi berjalan optimal, diikuti monitoring performa menggunakan alat seperti htop serta backup data terjadwal untuk menjaga integritas. Dengan langkah ini, sistem siap digunakan untuk mendukung operasional pemeliharaan ATM secara efisien.

  8. Maintenance

    9. Tahap maintenance Sistem Pemeliharaan ATM memastikan sistem tetap optimal, aman, dan stabil pasca-deployment. Pemantauan rutin dilakukan untuk menjaga performa dan stabilitas. Backup data dijadwalkan secara berkala untuk memastikan keamanan data. Keamanan diperkuat melalui pembaruan sistem, implementasi HTTPS, dan audit berkala. Penanganan bug dilakukan dengan memantau log error, pengujian berkala, serta feedback pengguna. Optimasi mencakup pembersihan data, optimasi database, caching, dan peningkatan kapasitas server jika diperlukan. Semua perubahan didokumentasikan dengan pelatihan kepada tim operasional agar sistem berjalan efektif dan mendukung proses secara berkelanjutan.
